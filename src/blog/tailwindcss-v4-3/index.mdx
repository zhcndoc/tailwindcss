Tailwind CSS v4.3 来了，而且显然，发布 v4.2 比起记得写博客要容易得多，所以这篇文章其实偷偷讲了两个版本的新内容。

以下是 v4.2 中新增的内容：

- [新的 `mauve`、`olive`、`mist` 和 `taupe` 调色板](#new-mauve-olive-mist-and-taupe-palettes) — 当 gray、zinc、neutral、stone 和 slate 不知为何都让你不满意时，再多四套偏中性的调色板。

- [新的 `font-features-*` 工具类](#new-font-features-utility) — 无需每次都编写自定义 CSS，就能控制底层 OpenType 特性。

……而以下是 v4.3 中新增的内容：

- [新的滚动条工具类](#new-scrollbar-utilities) — 无需纠结这周到底该用哪个半支持的浏览器 API，就能为滚动条设置样式。
- [新的 `@container-size` 工具类](#new-container-size-utility) — 在你的容器查询也需要关注高度时，用它来创建尺寸容器。
- [新的 `zoom-*` 工具类](#new-zoom-utilities) — 现在既然每个浏览器终于都同意它存在了，就可以直接在标记中使用 CSS `zoom` 属性。
- [新的 `tab-*` 工具类](#new-tab-utilities) — 控制制表符字符的渲染宽度，只是别像 GitHub 以前那样把它设成 8。
- [叠加 + 复合 `@variant` 支持](#stacked-compound-variant-support) — 可直接在 CSS 中使用叠加和复合变体。

## 一等公民的 webpack 插件

Tailwind CSS v4.2 新增了专用的 `@tailwindcss/webpack` 加载器，用于在 webpack 项目中集成 Tailwind CSS。

在此之前，webpack 项目通常通过 `postcss-loader` 和 `@tailwindcss/postcss` 运行 Tailwind，这虽然可行，但意味着 CSS 需要先通过 PostCSS 绕一小圈，Tailwind 才能拿到一个它本来就知道如何编译的字符串。

{

使用它的方式与你熟悉的其他 webpack loader 的设置方式是一样的：

```js
// [!code filename:webpack.config.js]
// [!code word:@tailwindcss/webpack]
const MiniCssExtractPlugin = require("mini-css-extract-plugin");

module.exports = {
  plugins: [new MiniCssExtractPlugin()],
  module: {
    rules: [
      {
        test: /\.css$/i,
        use: [MiniCssExtractPlugin.loader, "css-loader", "@tailwindcss/webpack"],
      },
    ],
  },
};
```

由于 Turbopack 通过其兼容层支持 webpack loaders，这些改进也会一并适用，这对 Next.js 这样的框架来说意义重大，因为 Turbopack 正在成为默认选项。

查看 [`@tailwindcss/webpack` README](https://github.com/tailwindlabs/tailwindcss/tree/main/packages/%40tailwindcss-webpack) 了解更多详情。

## 更多逻辑属性工具类

Tailwind CSS v4.2 新增了大量逻辑属性工具类，让构建能够在不同书写模式和方向下正确适配的布局变得更容易。

对于间距，现在有了用于 padding、margin、scroll padding、scroll margin 和 border 的 block-start 与 block-end 工具类：

```html
<!-- [!code classes:pbs-4,pbe-8,mbs-6,mbe-2] -->
<div class="mbs-6 mbe-2 pbs-4 pbe-8 ...">
  <!-- ... -->
</div>
```

```html
<!-- [!code classes:scroll-pbs-12,scroll-pbe-12,scroll-mbs-4,scroll-mbe-4] -->
<div class="scroll-mbs-4 scroll-mbe-4 scroll-pbs-12 scroll-pbe-12 ...">
  <!-- ... -->
</div>
```

```html
<!-- [!code classes:border-bs,border-be-2] -->
<div class="border-bs border-be-2 ...">
  <!-- ... -->
</div>
```

我们还为 `inline-size` 和 `block-size` 添加了逻辑尺寸工具类，以及最小和最大变体：

```html
<!-- [!code classes:inline-full,min-inline-0,max-inline-lg,block-64,min-block-24,max-block-screen] -->
<div class="block-64 inline-full max-block-screen max-inline-lg min-block-24 min-inline-0 ...">
  <!-- ... -->
</div>
```

此外，我们还为元素定位添加了逻辑 inset 工具类：

```html
<!-- [!code classes:inset-s-0,inset-e-4,inset-bs-2,inset-be-8] -->
<div class="absolute inset-s-0 inset-e-4 inset-bs-2 inset-be-8 ...">
  <!-- ... -->
</div>
```

现有的 `start-*` 和 `end-*` 工具类仍然可用，但它们现在已被弃用，建议改用 `inset-s-*` 和 `inset-e-*`，这样整个 API 就能与 `inset-bs-*` 和 `inset-be-*` 保持一致。

查看 [`padding`](/docs/padding#using-logical-properties)、[`margin`](/docs/margin#using-logical-properties)、[`scroll-padding`](/docs/scroll-padding#using-logical-properties)、[`scroll-margin`](/docs/scroll-margin#using-logical-properties)、[`border-width`](/docs/border-width#using-logical-properties)、[`inline-size`](/docs/inline-size)、[`block-size`](/docs/block-size) 以及 [`top / right / bottom / left`](/docs/top-right-bottom-left#using-logical-properties) 文档了解更多详情。

## 新的 `@container-size` 工具类

在 Tailwind CSS v4.0 中，我们通过 `@container` 工具类添加了官方的容器查询支持，它会创建一个 inline-size 容器。

这通常正是你想要的，但像 `cqb` 和 `cqh` 这样的容器查询长度单位依赖于容器的 block size，而 inline-size 容器不会暴露这类信息。

因此在 v4.3 中，我们添加了 `@container-size`，它会创建一个 size 容器：

```html
<!-- [!code word:@container-size] -->
<!-- [!code word:h-\[50cqb\]] -->
<div class="@container-size">
  <div class="h-[50cqb]">
    <!-- ... -->
  </div>
</div>
```

你也可以像 `@container/{name}` 一样，使用 `@container-size/{name}` 为 size 容器命名。

查看 [响应式设计文档](/docs/responsive-design#using-size-containers) 了解更多详情。

## 新的 `zoom-*` 工具类

Tailwind CSS v4.3 为 CSS `zoom` 属性添加了 `zoom-*` 工具类，这是那些古老的浏览器特性之一：早在 2008 年就已在 Chrome 1 中得到支持，但直到 2024 年，经过一些长期存在的 SVG bug 的曲折历程后，才在所有主流浏览器中全面支持：

```html
<!-- [!code classes:zoom-75,zoom-100,zoom-125] -->
<div class="zoom-75 ...">缩小</div>
<div class="zoom-100 ...">正常</div>
<div class="zoom-125 ...">放大</div>
```

你也可以使用任意值和 CSS 变量：

```html
<!-- [!code classes:zoom-[1.1],zoom-(--preview-zoom)] -->
<div class="zoom-[1.1] ...">稍微放大</div>
<div class="zoom-(--preview-zoom) ...">使用变量进行缩放</div>
```

查看 [`zoom`](/docs/zoom) 文档了解更多详情。

## 对堆叠式 + 复合 `@variant` 的支持

当你在 CSS 中而不是直接在标记中使用变体时，Tailwind CSS v4.3 让 `@variant` 更加灵活了。

现在你可以使用堆叠变体：

<CodeExampleStack>

```css
/* [!code filename:CSS] */
/* [!code word:hover\:focus] */
.button {
  background: var(--color-sky-500);

  @variant hover:focus {
    background: var(--color-sky-600);
  }
}
```

```css
/* [!code filename:Compiled CSS] */
.button {
  background: var(--color-sky-500);

  /* [!code highlight:8] */
  &:hover {
    @media (hover: hover) {
      &:focus {
        background: var(--color-sky-600);
      }
    }
  }
}
```

</CodeExampleStack>

而且你可以使用复合变体，在多个变体之间共享同一段 CSS：

<CodeExampleStack>

```css
/* [!code filename:CSS] */
/* [!code word:hover, focus] */
.button {
  background: var(--color-sky-500);

  @variant hover, focus {
    background: var(--color-sky-600);
  }
}
```

```css
/* [!code filename:Compiled CSS] */
.button {
  background: var(--color-sky-500);

  /* [!code highlight:6] */
  &:hover {
    @media (hover: hover) {
      background: var(--color-sky-600);
    }
  }

  /* [!code highlight:4] */
  &:focus {
    background: var(--color-sky-600);
  }
}
```

</CodeExampleStack>

通常更好的做法是创建一个真正的组件，并直接使用 Tailwind CSS 类进行样式设置，但当 CSS 才是合适的工具时，这会让 `@variant` 更好用很多。

查看 [`@variant`](/docs/adding-custom-styles#adding-custom-variants) 文档了解更多详情。

## 函数式工具类的默认值

Tailwind CSS v4.0 引入了函数式 `@utility` 定义，你可以使用 `--value(…)` 和 `--modifier(…)` 来定义接受主题值、裸值、任意值和修饰符的工具类。

在 v4.3 中，当某个工具类需要在有值和无值两种情况下都能工作时，你现在可以把 `--default(…)` 传给 `--value(…)` 和 `--modifier(…)`：

```css
/* [!code filename:CSS] */
/* [!code word:--default(4)] */
@utility tab-* {
  tab-size: --value(integer, --default(4));
}
```

现在，裸的 `tab` 工具类可以回退到 `4`，而 `tab-2` 仍然会解析为 `2`：

<CodeExampleStack>

```html
<!-- [!code filename:HTML] -->
<!-- [!code classes:tab,tab-2] -->
<pre class="tab ...">...</pre>
<pre class="tab-2 ...">...</pre>
```

```css
/* [!code filename:Generated CSS] */
.tab {
  /* [!code word:4] */
  tab-size: 4;
}

.tab-2 {
  /* [!code word:2] */
  tab-size: 2;
}
```

</CodeExampleStack>

这让定义 API 变得更容易：裸工具类执行显而易见的操作，同时仍然支持同一工具类家族中的显式值。

这虽是小事，但它让插件作者以及任何编写自定义工具类的人都能创建出更像我们在框架本身中提供的那些 API。
