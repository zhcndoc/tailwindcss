Tailwind Plus 中有许多 UI 模块需要 JavaScript 才能真正发挥作用，比如对话框、下拉菜单、命令面板等等。如果您不是 React 或 Vue 用户，以前使用这些 UI 模块就意味着必须自己编写所有那些复杂的 JavaScript 代码。

今天这一切终于改变了——**Tailwind Plus 中的每个 UI 模块现在都完全可用、无障碍且具备交互功能**，包括纯 HTML 示例。

现在，您可以在任何项目中使用任意 [下拉菜单](https://tailwindcss.com/plus/ui-blocks/application-ui/elements/dropdowns)、[命令面板](https://tailwindcss.com/plus/ui-blocks/application-ui/navigation/command-palettes)、[对话框](https://tailwindcss.com/plus/ui-blocks/application-ui/overlays/modal-dialogs)、[抽屉](https://tailwindcss.com/plus/ui-blocks/application-ui/overlays/drawers) 等，无需任何 JavaScript 框架。

---

## 无需框架

为实现这一点，我们构建了 `@tailwindplus/elements` —— 这是一个专门为 [Tailwind Plus](https://tailwindcss.com/plus) 用户发布的库。

Elements 是一组无界面（headless）的 [自定义元素](https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_custom_elements)，封装了构建交互式自定义 UI 需要的所有复杂行为，仅用 HTML 即可实现，样式可以用工具类或自定义 CSS 任意调整。

这些自定义元素不依赖特定的 JavaScript 框架，能在任何能使用 `<script>` 标签的地方运行：

```html
/* [!code filename:index.html] */
<script src="https://cdn.jsdelivr.net/npm/@tailwindplus/elements@1" type="module"></script>
```

为了支持 Tailwind Plus 中所有 UI 模块，这个 Elements 首个版本中包含了以下基础组件：

- [自动完成](https://tailwindcss.com/plus/ui-blocks/documentation/autocomplete)—— 用于构建自定义的 [组合框](https://tailwindcss.com/plus/ui-blocks/application-ui/forms/comboboxes)。
- [命令面板](https://tailwindcss.com/plus/ui-blocks/documentation/command-palette)—— 用于构建自定义的 [命令面板](https://tailwindcss.com/plus/ui-blocks/application-ui/navigation/command-palettes)。
- [对话框](https://tailwindcss.com/plus/ui-blocks/documentation/dialog)—— 用于自定义 [模态对话框](https://tailwindcss.com/plus/ui-blocks/application-ui/overlays/modal-dialogs)、[抽屉](https://tailwindcss.com/plus/ui-blocks/application-ui/overlays/drawers) 等。
- [可展开内容](https://tailwindcss.com/plus/ui-blocks/documentation/disclosure)—— 用于构建可折叠的 [常见问题](https://tailwindcss.com/plus/ui-blocks/marketing/sections/faq-sections#component-8699d80b13ef524eb573e54b4d4b89d1)、移动端菜单（在 [导航栏](https://tailwindcss.com/plus/ui-blocks/application-ui/navigation/navbars)中）等。
- [下拉菜单](https://tailwindcss.com/plus/ui-blocks/documentation/dropdown-menu)—— 用于构建自定义的 [下拉菜单](https://tailwindcss.com/plus/ui-blocks/application-ui/elements/dropdowns)。
- [弹出框](https://tailwindcss.com/plus/ui-blocks/documentation/popover)—— 用于构建自定义的 [飞出菜单](https://tailwindcss.com/plus/ui-blocks/marketing/elements/flyout-menus)等。
- [选择框](https://tailwindcss.com/plus/ui-blocks/documentation/select)—— 用于构建自定义的 [选择菜单](https://tailwindcss.com/plus/ui-blocks/application-ui/forms/select-menus#component-8298198b136afab3bb19391ae716077f)。
- [选项卡](https://tailwindcss.com/plus/ui-blocks/documentation/tabs)—— 用于构建自定义选项卡，就像我们在自定义 [多行文本框](https://tailwindcss.com/plus/ui-blocks/application-ui/forms/textareas#component-532d35bcb49664c16a4f877d71658b12)和[产品概览](https://tailwindcss.com/plus/ui-blocks/ecommerce/components/product-overviews#component-13a89c2dc50a31afd66541dc28fa3c13)中使用的那样。

如果您是 Tailwind Plus 用户，请访问全新的 [Elements 文档](https://tailwindcss.com/plus/ui-blocks/documentation/elements) 了解详细原理并查看示例。

## 利用现代 Web 技术

我们大量依赖现代平台特性，确保 Elements 轻量且尽可能原生高效。

- 使用 [自定义元素](https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_custom_elements) 作为跨平台组件抽象。
- 利用 [`popover` 属性](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Global_attributes/popover) 管理弹出层和弹出框，配合 [`beforetoggle`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLElement/beforetoggle_event) 控制过渡动画。
- 使用原生的 `<dialog>` 元素实现焦点锁定和顶层渲染。
- 通过 [Invoker 命令](https://developer.mozilla.org/en-US/docs/Web/API/Invoker_Commands_API) 声明式管理交互元素，比如切换自定义折叠内容。
- 利用 [`ElementInternals`](https://developer.mozilla.org/en-US/docs/Web/API/ElementInternals) 实现自定义表单控件与原生表单控件一样的行为。

我们还打包了这些特性所需的所有 polyfill，确保 Elements 能在所有 [Tailwind CSS v4.0 支持的浏览器](https://tailwindcss.com/docs/compatibility#browser-support)中正常运行。随着现代平台特性的普及，Elements 会变得更加轻量。
